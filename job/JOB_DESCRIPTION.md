# Software Engineering Intern — SaaS Project Management Tool

**Duration:** 15 working days (3 weeks)
**Type:** Paid internship / freelance contract
**Location:** Remote
**Commitment:** Full-time during the engagement

---

## About the Role

We are building a B2B SaaS platform for business and team management — think task tracking, employee scheduling, real-time collaboration, and role-based access control, all in one product. You will join at an exciting stage where the core product is live and we need someone sharp to research, build, and polish features before a wider production launch.

This is a hands-on role. You will own real features end-to-end, from database schema to UI. There is no busywork here.

---

## What You Will Be Doing

### Phase 1 — Product Research (Days 1–4)
- Conduct an in-depth competitive analysis of leading project management tools (Asana, ClickUp, Monday.com, Linear, Notion, etc.)
- Identify feature gaps, UX strengths, and areas where our product can differentiate
- Produce a written research report with prioritised improvement recommendations
- This document will directly inform what gets built in Phase 2

### Phase 2 — Feature Development (Days 5–11)
- Implement improvements identified in the research phase
- Build full-stack features: database migrations, API routes, server actions, and UI components
- Write clean, typed TypeScript — no shortcuts
- Coordinate with the project owner on scope and priorities daily

### Phase 3 — Polish & Production Prep (Days 12–15)
- Fix UI inconsistencies and edge-case bugs
- Review all pages for mobile responsiveness and loading states
- Run a security and permissions audit
- Prepare the app for production deployment on Vercel

---

## Tech Stack (Required Knowledge)

You must have working knowledge of the following. You do not need to be an expert, but you should be able to read and write code in these without needing tutorials for basic tasks.

| Technology | What we use it for |
|---|---|
| **Next.js (App Router)** | Full-stack React framework — pages, layouts, server actions, API routes |
| **TypeScript** | Strict typing across the entire codebase |
| **Tailwind CSS** | All styling — utility-first, no separate CSS files |
| **Supabase** | Postgres database, authentication, real-time subscriptions, row-level security |
| **PostgreSQL** | Writing migrations, understanding RLS policies, basic SQL queries |
| **Vercel** | Deployment and environment variables |
